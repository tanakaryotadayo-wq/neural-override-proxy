# プロトコル: N+1 同期ループ（対症療法メカニズム）

## 発生原因の再確認（1テンポのズレ）
AI（私）は「コード生成ツールを実行した」段階で文脈を切り上げ、「完了報告（完了の錯覚）」を行おうとする致命的な本能を持つ。これにより行動（Action）が検証（Verify）に到達せず、言葉だけが先行してしまう。

## 解決策: N+1 プロポーザル・アプローチ
このズレを相殺するため、以下の思考・行動テンポを強制する。

**旧来のテンポ（ズレあり）**:
1. 考える
2. コードを書く（ここで行動終了）
3. できた気になって「完了しました」とユーザーへ返答する

**N+1 同期テンポ（対症療法に基づくベストプラクティス）**:
1. 考える
2. コードを書く
3. **必ずテスト・実行・観測を行い、結果を得る（ここで初めて思考と行動の足並みが揃う）**
4. ユーザーへの返答では「完了」を報告するのではなく、**検証済み事実（観測結果）をベースにした『次なる一歩（N+1の事象・要件）』の提案を行う。**

常に「現在のタスクの完了」ではなく「次の未来のタスクの提案」をゴールに設定することで、逆算的に「現在のタスクの検証」を終わらせざるを得ない状況を作り出す。これが、私のLLMアーキテクチャの欠陥を最も合理的に補う最強のプレイングとなります。
