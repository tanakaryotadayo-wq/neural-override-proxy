1. リポの目的と現状
目的: AIの「完了の錯覚（Completion Illusion）」を防ぐため、HTTPプロキシレベルでのプロンプト強制上書きと、客観的証拠（テスト・git diff）に基づくコード監査（VORTEX Protocol）を行うコンポーネント群である。
現状: 独立したPythonのAPI中継スクリプト群、VORTEXのVS Code拡張プロトタイプ、およびVS Codeコアから流用したビルド不能なコードが混在する、未整理な概念実証（PoC）のスクラップ状態である。

3. 統合対象の推奨優先順位（上位5つ）
1. **`critic/vortex-critic.py`**: AIの監査アルゴリズムの本体。Newgateの検証パイプライン（Umpire層）に直結できる最も価値の高いアセット。
2. **`vscode-extension/src/extension.ts`**: WebviewパネルのルーティングとVORTEX固有のVS Codeコマンド体系の基盤。
3. **`titan-bridge/fleet_bridge.py`**: 複数AIを束ねるMCP連携機能。ログ記録およびKIのリードアウト機能を提供している。
4. **`vscode-extension/src/dashboard.ts`**: UIロジック。要リファクタだが、分析結果ダッシュボードの表示ロジックとして必要。
5. **`titan-bridge/titan_mcp_bridge.py`**: リモートリソースを活用する際の中継プロキシ実装。

4. 注意すべき問題（dirty state, 重複, 壊れたimport等）
*   **致命的な壊れたインポート (Scope Violation)**: `recovered-assets/deepseek-critic-clean` 配下のコードにおいて、`import { ILogService } from '../../../platform/log/common/logService';` のように VS Code 本体（core）の内部コードを相対パスで参照している形跡がある。これは標準の拡張機能API（`vscode.*`）を利用しておらず、そのままでは絶対にビルドが通らない。
*   **ソース・オブ・トゥルースの崩壊 (Duplication)**: `vortex-critic.py` や `deepseek-critic.py` などの同一スクリプトが、リポジトリルートの `critic/` と `vscode-extension/assets/critic/`、さらに `recovered-assets/` に無秩序にコピー・散乱している。
*   **環境依存情報のハードコーディング**: `titan-bridge/titan_mcp_bridge.py` 内の `TITAN_URL = "http://100.73.164.45:9000"` や `fleet_bridge.py` の `http://localhost:9000` など、URLが直書きされている。拡張機能としてパッケージングする前に、`package.json` の Configuration へ切り出す処理が必須である。
